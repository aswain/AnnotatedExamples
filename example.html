Problems Domain

Key Components

Problem Section

Container for information related to the patient's clinical problems. Narrative text is captured within the section and structured data are contained within coded entries.

Example of Problems Section

In this example, the Problems Section reflects the patient's problem list with active problems listed separately from historical problems. Coded data representing the problem list entry is referenced in the narrative directly from the entry.

<component>
	<section>
		<!--  *** Problems section with entries required *** -->
		<templateId root="2.16.840.1.113883.10.20.22.2.5.1"/>
		<code code="11450-4" codeSystem="2.16.840.1.113883.6.1" codeSystemName="LOINC"
			displayName="PROBLEM LIST"/>
		<title>PROBLEMS</title>
		<text>
			<paragraph>Active Concerns</paragraph>
			<list>
				<item>Problem #1<list>
						<item><content ID="problem1">Chest pain</content>onset Apr 14, 2007</item>
						<item><content ID="problem2">Angina</content>onset Apr 17, 2007</item>
					</list>
				</item>
			</list>
			<paragraph>Resolved Concerns</paragraph>
			<list>
				<item>Problem #2<list>
						<item><content ID="problem3">Pneumonia - Left lower lobe</content>onset Mar 10, 1998; resolution Mar 16, 1998</item>
					</list>
				</item>
			</list>
		</text>
		<entry typeCode="DRIV"> ...
	</section>
</component>

Problem Concern Act

Defines business rules constraining the Act Clinical Statement to represent the concern. Related problem observations are nested within the Act wrapper to allow for tracking over time.

Example of Problem Concern Act

In this example, the Problem Concern Act contains multiple problem observations related to the same concern being tracked.

<act classCode="ACT" moodCode="EVN">
	<!-- Problem concern act template -->
	<templateId root="2.16.840.1.113883.10.20.22.4.3"/>
	<id root="ec8a6ff8-ed4b-4f7e-82c3-e98e58b45de7"/>
	<code code="CONC" codeSystem="2.16.840.1.113883.5.6" displayName="Concern"/>
	<statusCode code="active"/>
	<effectiveTime>
		<low value="20070103"/>
	</effectiveTime>
	<entryRelationship <typeCode="DRIV"/>
		<observation classCode="OBS" moodCode="EVN">
			<!-- Problem observation template -->
			<templateId root="2.16.840.1.113883.10.20.22.4.4"/>
			...
		</observation>
	</entryRelationship>
</act>

Problem Observation

Defines business rules constraining the Observation Clinical Statement to represent the problem observed. Problem Observations are contained within the Problem Concern Act, which represents the overarching concern being tracked.

Example of Problem Observation

In this example, the Problem Observation contains a code representing the problem observed as well as qualifiers to precisely describe the problem.

<observation classCode="OBS" moodCode="EVN">
	<!-- Problem observation template -->
	<templateId root="2.16.840.1.113883.10.20.22.4.4"/>
	<id root="ab1791b0-5c71-11db-b0de-0800200c9a66"/>
	<code code="409586006" codeSystem="2.16.840.1.113883.6.96"
		displayName="Complaint"/>
	<text>
		<reference value="#problem2"/>
	</text>
	<statusCode code="completed"/>
	<effectiveTime>
		<low value="20070103"/>
	</effectiveTime>
	<value xsi:type="CD" code="195967001" codeSystem="2.16.840.1.113883.6.96"
		displayName="Asthma"/>
	<entryRelationship typeCode="SUBJ"
		inversionInd="true">
		<observation classCode="OBS" moodCode="EVN">
			<templateId root="2.16.840.1.113883.10.20.22.4.31"/>
			<!-- Age observation template -->
			...
		</observation>
	</entryRelationship>
	<entryRelationship typeCode="REFR">
		<observation classCode="OBS" moodCode="EVN">
			<templateId root="2.16.840.1.113883.10.20.22.4.5"/>
			<!-- Health status observation template -->
			...
		</observation>
	</entryRelationship>
</observation>

Problem Status

Defines business rules constraining the Observation Clinical Statement to represent the current status of the problem. Problem Status is contained within the Problem Observation to indicate if the problem is active, inactive, or resolved.

Example of Problems Status

In this example, the Problem Status indicates that this problem is known to be resolved.

<observation classCode="OBS" moodCode="EVN">
  <!-- Status observation template -->
  <templateId root="2.16.840.1.113883.10.20.22.4.6"/>
  <code code="33999-4" 
        codeSystem="2.16.840.1.113883.6.1"
        codeSystemName="LOINC" 
        displayName="Status"/>
  <text>
	   <reference value="#STAT1"/>
  </text>
  <statusCode code="completed"/>
  <value xsi:type="CD" 
         code="413322009"
         codeSystem="2.16.840.1.113883.6.96"
         codeSystemName="SNOMED CT" 
         displayName="Resolved"/>
</observation>
